  - `yearsPerTurn` (integer, required): years passing per turn during this interval.
  - `untilTurn` (integer, required): the turn at which this interval ends.

  The last `untilTurn` value is ignored for year calculation after that point â€” the final `yearsPerTurn` continues indefinitely. However, it is used as the reference for `Era.startPercent`.

  ```json
      {"yearsPerTurn": 40, "untilTurn": 80}
  ]
  ```

  In this example the first 50 turns last 60 years each, then turn 51 onward lasts 40 years each.

<Note>

The `constants` sub-object is the only structure **merged field-by-field** across mods rather than overwritten â€” this allows different mods to change different constants without conflict. #### UnitUpgradeCost

Nested in `constants.unitUpgradeCost`. The entire sub-structure is replaced when overridden (fields are not individually merged).

| Attribute | Type | Default | Notes |
|-----------|------|---------|-------|
| `base` | Float | 10 | Base cost component |
| `perProduction` | Float | 2 | Cost per point of production difference between old and new unit |
| `eraMultiplier` | Float | 0 | Era-scaling multiplier |
| `exponent` | Float | 1 | Exponent applied to the result |
| `roundTo` | Integer | 5 | Cost is rounded down to a multiple of this |

Formula (rounded down to a multiple of `roundTo`):

```
(max(base + perProduction Ã— (newUnitCost âˆ’ oldUnitCost), 0)
  Ã— (1 + eraNumber Ã— eraMultiplier)
  Ã— civModifier) ^ exponent
```

`civModifier` is the product of all applicable `[relativeAmount]% Gold cost of upgrading` uniques.

<Accordion title="GlobalUniques.json">

Defines uniques that apply globally. In the Vanilla ruleset this is where the effects of Unhappiness are defined.

<Note>

<Accordion title="Tutorials.json">

Defines tutorial entries shown to players. A base ruleset mod can add a **welcome page** by defining a tutorial whose `name` matches the mod name.

<Note>
  Unlike most JSON files in a base ruleset mod, `Tutorials.json` **adds to** the default tutorials rather than replacing them. Place it at `<mod>/jsons/` even though the original file lives one level above the Vanilla jsons.

  If a tutorial popup has an image under `ExtraImages/` with the same name, it is shown in the popup. This lookup is case-sensitive on some platforms. This feature requires Unciv version 4.11.5 or later.
</Note>

<ParamField path="name" type="string" required>
  Entry name. Setting this equal to the mod name creates a welcome page for the mod.
</ParamField>

<ParamField path="category" type="string">
  Category where this tutorial is displayed. Defaults to `"Tutorials"`.
</ParamField>

<ParamField path="civilopediaText" type="object[]">
  Extra formatted text. Shown before `steps`. See the [Civilopedia text](#civilopedia-text) section.
</ParamField>

<ParamField path="steps" type="string[]">
  Plain-text tutorial steps shown in the tutorial popup.
</ParamField>

</Accordion>

## Civilopedia text

Any game object defined in JSON and listed in the Civilopedia can supply extra text via a `civilopediaText` field. This can explain special considerations or provide flavour content when auto-generated display is insufficient.

### Example

```json
"civilopediaText": [
    { "text": "Ancient ruins provide a one-time random bonus when explored" },
    { "separator": true },
    {
        "text": "This line is red and links to the Scout including icons",
        "link": "Unit/Scout",
        "color": "red"
    },
    {
        "text": "A big fat header sporting a golden star",
        "header": 1,
        "starred": true,
        "color": "#ffeb7f"
    }
]
```

### Attributes

Not all combinations are valid.

| Attribute | Type | Description |
|-----------|------|-------------|
| `text` | String | Text to display. Supports inline color markup with `Â«colorÂ»` to start and `Â«Â»` to stop. |
| `link` | String | Creates a link and icon. Format: `Category/Name` or an external URL (`http://`, `https://`, `mailto:`). |
| `icon` | String | Shows an icon without a link. Format: `Category/Name`. |
| `extraImage` | String | Displays an image instead of text. Can be a texture atlas path or a name from the `ExtraImages` folder. |
| `imageSize` | Float | Size in world units for `extraImage`. The smaller dimension is calculated to preserve aspect ratio. |
| `header` | Integer | Header level. `1` is double text size; higher values decrease size from there. |
| `size` | Integer | Text size. Use `size` or `header` but not both. Default is 18. |
| `indent` | Integer | Indent level. `0` follows icons; `1` aligns right of all icons; each further step is 30 units. |
| `padding` | Float | Vertical padding between rows. Default is 5 units. |
| `color` | String | Text color. Accepts color names or 3/6-digit web hex codes (e.g. `#FFA040`). |
| `separator` | Boolean | Renders a separator line instead of text. Can only be combined with `color` and `size` (line width, default 2). |
| `starred` | Boolean | Decorates text with a star icon. The star receives the `color` instead of the text. |
| `centered` | Boolean | Centers the line and disables automatic wrap. For `extraImage`, crops to content to equalize transparent borders. |
| `iconCrossed` | Boolean | Paints a red X over the `icon` or `link` image. |

JSON lines surround the auto-generated lines such that the latter are inserted just above the first JSON line carrying a `link`. If no JSON lines have links, they are inserted between the automatic title and the automatic info.
